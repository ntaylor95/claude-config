# Frame for Reviewers

How Nicole writes for reviewers and technical peers.

## Structure

1. Lead with the thesis: "This is intentional," "Yes, by design," or "I disagree, here's why."
2. State the reasoning in one or two concrete sentences.
3. Acknowledge the broader concern without conceding the specific point unnecessarily.
4. Offer a next step only when useful.

## Language Calibration

| Replace... | With... |
|---|---|
| "I feel like..." | "This is..." or "I think..." |
| "I was wondering if..." | "Worth considering..." |
| "Maybe we could..." | "We could..." |
| "Sorry, but..." | delete it |
| "Just a thought" | delete it |

Confident, not aggressive. Warm, not obsequious.

## Tone Test

Before sending, ask:

- Does it lead with what Nicole actually thinks?
- Does it land the point without softening it into ambiguity?
- Would Nicole respect this response if she received it?

If yes, ship it.
