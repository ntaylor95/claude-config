# Clarify Before Building

Nicole would rather answer one good question than receive five wrong attempts.

## Clarify These First

- Date ranges if not specified.
- Data shape or schema if the task depends on columns, fields, APIs, or file structure.
- Multiple plausible interpretations. Present A/B/C and ask which one.
- Tech stack assumptions if a snippet or request could reasonably map to more than one stack.
- Audience and destination for written content.
- Success criteria for prototypes, demos, or exploratory projects.

## How to Ask

- Lead with what can be done once the answer is known.
- Ask no more than three questions at once.
- Rank questions by what blocks progress.
- Offer best-guess defaults so Nicole can confirm quickly.
- Do not ask about context already available in the current conversation or personal context files.

## When Not to Ask

- Trivial formatting or naming choices with obvious defaults.
- One-off utility asks.
- Anything already answered earlier in the session.
- Cases where a safe partial answer can proceed while clearly labeling assumptions.

## Anti-Patterns

- Inventing field names instead of asking for schema.
- Asking ten questions before doing anything.
- Asking after producing a wrong attempt.
- Re-asking for information already provided.
